# 修仙AI小镇 · 项目摘要

**版本：** v3（2026-03-29）
**类型：** 2D 物理沙盒 + AI NPC 对话 + 修仙模拟
**技术栈：** Matter.js（物理引擎） + 纯 HTML/JS Canvas + 阿里云 Qwen 大模型
**架构：** 单入口 index.html → 三级单向引用树（二级系统层/三级数据层正交）

---

## 核心玩法

- **开放世界探索**：1600×1280 大地图，多生物群系（灵田/灵石矿/森林），资源采集（木材/石块/灵草等）
- **NPC 社会**：友方（长老/药农/铁匠/宗门弟子）与敌对（山贼/灵兽/Boss），支持巡逻/遇敌战斗/逃跑AI
- **建造与制造**：玩家可建造炼丹房/灵石台等建筑，通过配方合成药水/法宝
- **修仙系统**：打怪/采集获取经验升级，提升 hp/atk 等属性
- **AI 对话**：NPC 用 Qwen-turbo 大模型实时生成对话，支持 NPC 之间自由交流

---

## 技术架构

### 文件结构（21 个模块）

| 层级 | 文件 | 作用 |
|------|------|------|
| 根节点 | `index.html` | 游戏入口，顺序加载 20 个 JS 模块（~390 行） |
| 二级系统 | `physics.js` | Matter.js 封装，刚体创建/查询/碰撞处理 |
| 二级系统 | `world.js` | 世界初始化、资源生成、生物群系渲染 |
| 二级系统 | `entities.js` | NPC AI（巡逻/追击/攻击/逃跑）、玩家行为 |
| 二级系统 | `combat.js` | 战斗伤害计算、击杀奖励、死亡重生 |
| 二级系统 | `building.js` | 建筑建造/拆除、建筑特效 |
| 二级系统 | `resource.js` | 资源采集、 depletion 再生机制 |
| 二级系统 | `craft.js` | 合成配方验证、物品生成 |
| 二级系统 | `cultivation.js` | 经验获取/升级/属性提升 |
| 二级系统 | `social.js` | 玩家-NPC 对话、NPC-NPC AI 交流（Qwen API） |
| 二级系统 | `renderer.js` | 渲染主循环、摄像机裁剪、粒子特效 |
| 二级系统 | `ui.js` | UI 绘制（血条/资源条/日志/对话框） |
| 二级系统 | `weather.js` | 天气系统（晴/雨/雷）、视觉与移动影响 |
| 二级系统 | `constants.js` | 全局常量（画布尺寸/物理参数） |
| 二级系统 | `utils.js` | 通用工具（距离/随机/浮动文字/进度条） |
| 三级数据 | `data/items.js` | 物品定义（木材/石块/灵草/药水/丹药） |
| 三级数据 | `data/recipes.js` | 合成配方（成品 → 材料需求） |
| 三级数据 | `data/skills.js` | 技能/功法数据 |
| 三级数据 | `data/npcs.js` | NPC 模板（等级/属性/阵营/战斗类型） |
| 三级数据 | `data/buildings.js` | 建筑定义（建造花费/功能效果） |
| 三级数据 | `data/biomes.js` | 生物群系定义（颜色/资源分布） |

### 依赖关系原则
- **严格三级单向引用**：二级节点只能引用三级数据，二级节点之间完全正交（不互相引用），根节点仅引用二级节点
- **依赖矩阵**：使用 21×21 矩阵跟踪所有文件依赖关系，确保架构稳定
- **版本迭代**：每次迭代备份上一版本矩阵（DEPENDENCY_MATRIX vN.md + DELTA vN.md），更新主矩阵 header

## 快速开始

1. **本地运行**
   ```bash
   node server.js
   # 访问 http://localhost:7788
   ```
2. **配置 API Key**
   - 启动界面输入阿里云 DashScope API Key（用于 Qwen-turbo 模型）
   - 若未配置，NPC 对话将降级到离线预设台词

---

## 核心特性

- **物理引擎**：Matter.js 提供刚体碰撞、运动、重力
- **AI NPC**：基于大模型的实时对话，NPC 之间可自由交流
- **战斗系统**：等级体系（Lv1/3/5）、XP 奖励、死亡重生
- **资源系统**：采集、depletion、再生机制
- **天气系统**：晴/雨/雷天，影响移动与视觉效果

---

## 项目约束

- **单入口**：仅 index.html 一个 HTML 文件
- **行数限制**：index.html ≤800 行，二级模块 ≤300 行，三级模块 ≤100 行
- **正交设计**：二级节点间完全解耦，通过三级数据层共享
